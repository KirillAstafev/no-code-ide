# План реализации: Генератор Java-кода для Spring Boot-приложения

## Контекст

Проект no-code-ide - это визуальный редактор схем с использованием библиотеки `@gravity-ui/graph`. В проекте есть модули, источники данных и приёмники. Есть существующая функция `buildProject`, которая скачивает Spring Boot шаблон и выполняет сборку.

## Требования

1. **Генерация Java-кода**: Добавить генерацию исходного кода Java на основе данных проекта (project.json, schema, файлы модулей)
2. **Сохранение в папку generated**: Код должен сохраняться в `/generated/имя_сгенерированного_приложения`
3. **Распределение по пакетам**: Каждый модуль должен быть в отдельном пакете (например, `com.example.имя_проекта.имя_модуля`)
4. **Классы для связей**: Каждая связь источник_данных -> модуль должна быть представлена отдельным Java-классом с аннотацией `@Component`
5. **Заглушки**: Вместо вызова команд для источников и приёмников использовать `System.out.println` с названием команды

## Анализ текущей структуры

### Типы данных (`src/types/model.d.ts`)

```typescript
interface Project {
    name: string;
    location: string;
    modules: Module[];
    sources: DataSource[];
    destinations: DataDestination[];
    dependencies: ExternalDependency[];
    schema: Schema;
}

interface Module {
    name: string;
    location: string;
    sources: DataSource[];  // Источники данных для этого модуля
    destinations: DataDestination[];  // Приёмники данных для этого модуля
}

interface DataSource {
    name: string;
    ipAddress: string;
    tcpPort: number;
    command?: DataSourceCommand;
    commandParams?: Record<string, string | number | boolean>;
}

interface DataDestination {
    name: string;
    url: string;
    dependency: ExternalDependency;
}

interface DataSourceCommand {
    name: string;
    description: string;
    parameters: DataSourceCommandParameter[];
}
```

### Структура связей (из schema.edges)

Связи между источниками и модулями определяются по `sourceBlockId` и `targetBlockId` в `schema.edges`.

### Существующая функция `buildProject`

Функция уже реализована в `src/electron/utils/project.ts` и:
1. Создаёт папку `generated`
2. Скачивает Spring Boot шаблон с start.spring.io
3. Распаковывает шаблон
4. Выполняет Gradle сборку

Нужно расширить эту функцию для генерации Java-кода.

### Шаг 4: Генерация Java-файлов для связей

**Цель**: Создать отдельные классы-сервисы для каждой связи источник_данных -> модуль.

**Пример генерируемого кода для связи**:

```java
package com.example.имя_проекта.имя_модуля;

import org.springframework.stereotype.Component;

@Component
public class Источник1Client {
    
    private static final String SOURCE_NAME = "Источник1";
    private static final String IP_ADDRESS = "192.168.1.100";
    private static final int TCP_PORT = 8080;
    
    public String executeCommand(String commandName) {
        System.out.println(commandName + " - вызов команды для источника " + SOURCE_NAME);
        return "Результат выполнения команды";
    }
    
    public String executeCommandWithParams(String commandName, java.util.Map<String, Object> params) {
        System.out.println(commandName + " - вызов команды для источника " + SOURCE_NAME + " с параметрами");
        return "Результат выполнения команды с параметрами";
    }
}
```

## Примерная структура изменений в project.ts

```typescript
import {generateJavaProject, generateJavaFiles} from './javaGenerator.js';

export const buildProject = async (
    event: IpcMainInvokeEvent,
    project: Project
): Promise<{ success: boolean; path?: string; error?: string }> => {
    try {
        const projectDir = project.location;
        const generatedDir = path.join(projectDir, 'generated');
        
        // Удаляем старую папку generated
        try {
            await fs.rm(generatedDir, { recursive: true, force: true });
        } catch (err) {
            // Игнорируем ошибки при удалении несуществующей папки
        }

        await fs.mkdir(generatedDir, { recursive: true });
        
        // 1. Скачиваем Spring Boot шаблон (существующий код)
        // ...
        
        // 2. Генерируем Java-код на основе данных проекта (НОВЫЙ КОД)
        const generatedProjectInfo = generateJavaProject(project);
        await generateJavaFiles(project, generatedProjectInfo, generatedDir);
        
        // 3. Выполняем сборку (существующий код)
        // ...
        
        return { success: true, path: generatedDir };
    } catch (error) {
        console.error('Ошибка сборки проекта:', error);
        return { success: false, error: (error as Error).message };
    }
};
```

# План реализации: Сохранение и отображение команд в связях модулей

## Контекст

Проект no-code-ide - это визуальный редактор схем с использованием библиотеки `@gravity-ui/graph`. В проекте есть модули, источники данных и приёмники. При настройке связей модуля открывается `ModuleConnectionsModal`, где можно выбрать источники и настроить для них команды (с параметрами).

## Требования

1. **При сохранении связей**: в схеме (schema.edges) должна сохраняться информация о настроенной команде (название + параметры) для каждой связи между модулем и источником данных.
2. **При отображении на холсте**: на метке соединения должна отображаться название настроенной команды.

## Анализ текущей структуры

### Типы данных (`src/types/model.d.ts`)

```typescript
interface SchemaEdge {
    id: string;
    sourceBlockId: string;

    label?: string;  // <-- ЭТО ПОЛЕ БУДЕМ ИСПОЛЬЗОВАТЬ ДЛЯ НАЗВАНИЯ КОМАНДЫ
    style?: 'solid' | 'dashed';
    arrowhead?: 'none' | 'arrow' | 'dot';

```

## Технические детали

### Сохранение команды в модуле

При сохранении связей в `Editor.tsx` мы обновляем источник данных в модуле, добавляя поля `command` и `commandParams`:

```typescript
sources: sourceConnections.map(sc => {
    const source = project.sources?.find(s => s.name === sc.sourceName);
    if (source) {
        const command = DATA_SOURCE_COMMANDS.find(c => c.name === sc.commandName) || DATA_SOURCE_COMMANDS[0];
        const updatedSource: DataSource = {
            ...source,
            command: command,
            commandParams: sc.commandParams
        };
        return updatedSource;
    }
    return null as any;
}).filter((source): source is DataSource => source !== null)
```

### Сохранение команды в схеме

При сохранении проекта в `project.ts` метки соединений сохраняются из схемы:

```typescript
edges.push({
    id: `edge-${sourceId}-to-${moduleId}`,
    sourceBlockId: sourceId,
    targetBlockId: moduleId,
    label: source.command?.name || 'command',  // <-- Название команды
    arrowhead: 'arrow',
});
```
